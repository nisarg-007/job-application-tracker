Original Table (1NF)

📌 What is 1NF?

All data is stored in a single flat table. No repeating groups. Every cell holds exactly one value.

Original Relation: BOOK_ROYALTY (ISBN, BookTitle, Author_Name, LastName, Publisher, Royalty, Edition)
Primary Key: ISBN (single attribute — underlined in diagrams)

  • DIRECT ISBN → BookTitle, Author_Name, LastName, Publisher, Royalty, Edition (ISBN determines everything)
  • TRANSITIVE Author_Name → LastName  (non-key → non-key, violates 3NF)
  • TRANSITIVE Publisher → Royalty  (non-key → non-key, violates 3NF)

⚠️ Why is this only 1NF?
The table has a single-attribute PK (ISBN) — so there are NO partial dependencies (you can only have partial deps with a composite key). However, it has two transitive dependencies: ISBN → Author_Name → LastName and ISBN → Publisher → Royalty. These violate 3NF but not 2NF.

2NF — Already Satisfied

📌 What is 2NF?

A table is in 2NF when it is in 1NF AND every non-key attribute is fully functionally dependent on the entire primary key (no partial dependencies).

Because our PK is a single attribute (ISBN), it is impossible to have a partial dependency — every attribute must depend on ISBN (the whole key). Therefore the original table is already in 2NF.

No decomposition is required for 2NF.
2NF — TABLE: BOOK_ROYALTY  (Unchanged — Already in 2NF)
PK: ISBN  |  ✅ No partial dependencies — single-attribute PK means every attribute depends on the whole key.
⚠️ Dashed orange arcs = transitive dependencies still present (Author_Name→LastName, Publisher→Royalty) — these violate 3NF, not 2NF.
Key rule: Partial dependencies can only occur when the primary key is composite (multi-attribute). Since ISBN alone is the PK here, the table satisfies 2NF automatically. The transitive dependencies are a 3NF violation, not a 2NF violation.

3NF — Remove Transitive Dependencies

📌 What is 3NF?

A table is in 3NF when it is in 2NF AND no non-key attribute is transitively dependent on the primary key — meaning no non-key attribute determines another non-key attribute.

Two transitive chains to break:
  • TRANSITIVE ISBN → Author_Name → LastName  →  Extract AUTHOR(Author_Name, LastName)
  • TRANSITIVE ISBN → Publisher → Royalty  →  Extract PUBLISHER(Publisher, Royalty)

Result: 3 tables — BOOK, AUTHOR, PUBLISHER
